Getting this blog underway with Jekyll

Sep 10, 2015

This blog is the child of many different mini blog creations until I found a good format. I've settled (or succeeded) with Jekyll. I found so many nice wordpress templates and even taught myself a decent amount of PHP to connect to and manage my posts with MySQL.

After all of my research and learning, I've decided to go with jekyll because it is free! The downside is that all of the PHP and MySQL that I just learned means nothing because git hosted pages cannot process them :-( But I need them for future jobs (maybe?). I'm just starting to work in technology and honestly don't have a lot of money so really this is an economic decision more than anything. If i'm rolling in it in a year's time, then maybe I'll make my own fancy WP page hosted through a fancy server

Also, this article from found To add new posts, simply add a file in the _posts directory that follows the convention YYYY-MM-DD-name-of-post.ext and includes the necessary front matter. Take a look at the source for this post to get an idea about how it works.

Jekyll also offers powerful support for code snippets:

def print_hi(name)
  puts "Hi, #{name}"
end
print_hi('Tom')
#=> prints 'Hi, Tom' to STDOUT.

Check out the Jekyll docs for more info on how to get the most out of Jekyll. File all bugs/feature requests at Jekyll’s GitHub repo. If you have questions, you can ask them on Jekyll’s dedicated Help repository.
